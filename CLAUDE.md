## Project Overview

`cb-cc-bot` is a Go daemon that automates a BTC → USDC market sell on Coinbase Advanced Trade whenever a Coinbase credit card autopay reminder email arrives in Gmail. The goal is to pre-fund the USDC balance before the autopay executes, avoiding Coinbase Simple Trade's higher fees.

**Flow:**
1. On startup, record `time.Now()` in memory as the Gmail watermark — only emails arriving after this point are processed
2. Poll Gmail every 5 minutes for a Coinbase autopay reminder email (subject: "Reminder - Your automatic payment is coming up", from: noreply@creditcard.coinbase.com)
3. Parse the USD amount from the body: `Your automatic payment of $X is scheduled for...`
4. Check current USDC balance on Coinbase Advanced Trade
   - If balance >= payment: send Telegram notification that no sell is needed, done
   - If balance < payment: `sell_amount = payment - usdc_balance`
5. Send Telegram approval prompt showing full math, with Yes/No inline keyboard
6. On "Yes": place a market sell on Coinbase Advanced Trade (BTC-USDC) with `quote_size = sell_amount`
7. On "No": send Telegram notification that the sell was skipped

## Deployment

Runs on a personal Kubernetes cluster as a `Deployment` with `replicas: 1`. No PVC required (no persistent state). All secrets are mounted from Kubernetes Secrets.

## Commands

```bash
go build ./...            # build all packages
go run cmd/bot/main.go    # run the bot
go run cmd/auth/main.go   # one-time Gmail OAuth2 browser flow — generates token.json locally
go test ./...             # run all tests
go vet ./...              # static analysis
```

## Architecture

```
cmd/bot/main.go       # entry point: loads config, wires deps, starts Gmail poll loop + Telegram getUpdates loop
cmd/auth/main.go      # one-time OAuth2 helper: opens browser, writes token.json
internal/config/      # config struct populated from environment variables
internal/gmail/       # Gmail API client: OAuth2 setup, poll using after:<epoch> query, body extraction
internal/email/       # parse USD payment amount from Coinbase reminder email body
internal/coinbase/    # Coinbase Advanced Trade REST client: USDC balance, BTC ticker, market sell order
internal/telegram/    # Telegram Bot API: send messages, inline keyboard, long-poll getUpdates for callbacks
internal/bot/         # orchestration: gmail → email → coinbase balance check → telegram → order execution
```

**Key design notes:**
- **No persistent state**: Gmail watermark is `time.Now()` at startup, passed as `after:<unix_epoch>` in the search query. Pod restarts simply advance the watermark — emails that arrived during downtime are intentionally ignored.
- **One active prompt at a time**: If a second reminder email arrives while a Telegram approval is pending, it is skipped with a log warning.
- **Market sell sizing**: Order uses `quote_size` in USD so Coinbase handles exact BTC quantity. BTC estimate shown in the Telegram prompt is from the current ticker and is informational only.
- **USDC balance offset**: Sell amount is `max(0, payment_amount - usdc_balance)`. Telegram message always shows the full math.

## Configuration (environment variables)

| Variable | Description |
|---|---|
| `GMAIL_CREDENTIALS_FILE` | Path to `credentials.json` from Google Cloud Console (default: `credentials.json`) |
| `GMAIL_TOKEN_FILE` | Path to `token.json` generated by `cmd/auth` (default: `token.json`) |
| `COINBASE_API_KEY_NAME` | Coinbase CDP API key name (format: `organizations/.../apiKeys/...`) |
| `COINBASE_PRIVATE_KEY` | Coinbase CDP EC private key PEM string |
| `TELEGRAM_BOT_TOKEN` | Telegram bot token from BotFather |
| `TELEGRAM_CHAT_ID` | Your personal chat ID with the bot |
| `POLL_INTERVAL` | Gmail poll interval (default: `5m`) |

## Coinbase Advanced Trade API

- Auth: CDP API keys using ES256 JWT (not HMAC). `COINBASE_API_KEY_NAME` is the key name; `COINBASE_PRIVATE_KEY` is the EC private key PEM.
- Base URL: `https://api.coinbase.com/api/v3/brokerage`
- Endpoints used:
  - `GET /accounts` — find USDC account available balance
  - `GET /best_bid_ask?product_ids=BTC-USDC` — current BTC ask price (for display estimate only)
  - `POST /orders` — `order_configuration.market_market_ioc.quote_size: "<usd>"`, `side: SELL`, `product_id: BTC-USDC`

## Telegram Bot

- Approval prompt uses inline keyboard with `callback_data: approve` / `callback_data: reject`
- `getUpdates` long-polls with 30s timeout in a separate goroutine to receive button callbacks
- Both the Gmail poll loop and Telegram loop run concurrently; `bot.pendingAmount` is protected by a mutex
